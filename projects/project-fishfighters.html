Fish Fighters

Personal Project – C++ | SFML 3.0 | Custom 2D Engine

Fish Fighters started as a personal challenge to learn how to structure a larger game project. I wanted to move beyond short school assignments and explore what it means to write clean, modular, and maintainable code for a real game. The result is a tower defense prototype inspired by the mobile game The Battle Cats: simple enough to build alone, yet complex enough to test architecture, systems, and workflows.

Project Goal

The primary goal was not to build a highly complex game, but to understand how to design a proper codebase. I focused on clean architecture, clear separation of responsibilities, and a level of modularity that would allow the game to evolve without rewriting everything each time.

A tower defense game was a perfect fit. It is achievable on my own, but still involves enough systems to create real engineering challenges.

What I Built

I developed everything myself using C++20 and SFML 3.0. The project helped me implement several essential systems for 2D game development.

  • A state machine for all battle entities (BaseState → AttackState, WalkingState, IdleState, etc.)
  • A custom animation system using TextureRects to scroll through spritesheets
  • A modular UI framework (UIElement → UIButton, UITexture, UIText) handled by a UIManager
  • A DataLoader powered by nlohmann::json to load units, enemies, levels, and player saves
  • A level system managing enemy waves and battle flow
  • A built-in debug UI using ImGui (spawn entities, give resources, load levels, adjust stats)
  • Tweeny integration for quick, smooth UI and gameplay animations

One of the results I am most proud of is the modularity: I can add a new enemy, unit, or level simply by editing a JSON file. No code changes required. That was a core objective from the start, and it works beautifully.

Technologies

  • C++20
  • SFML 3.0
  • ImGui
  • Tweeny
  • Nlohmann JSON
